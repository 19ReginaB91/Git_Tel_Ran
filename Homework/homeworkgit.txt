Чтобы отказаться от изменений в файле (файлах) находящихся в рабочем каталоге, можно использовать команду:
lua

git checkout -- <file>
где <file> - путь к файлу или файлам, которые нужно отменить.

Чтобы вернуть файл из индекса в рабочий каталог, можно использовать команду:
perl

git reset HEAD <file>
где <file> - путь к файлу, который нужно вернуть.

Чтобы отменить ошибочный commit, можно использовать команду:
php

git revert <commit>
где <commit> - идентификатор коммита, который нужно отменить.

Чтобы создать новую ветку и переключиться в нее, можно использовать команду:
css
git checkout -b <branch>
где <branch> - название новой ветки.

Fast-forward merge является простым и быстрым типом слияния веток, при котором все изменения из одной ветки просто добавляются в другую ветку без создания дополнительного коммита слияния. Non fast-forward merge, в отличие от fast-forward merge, создает новый коммит слияния даже в случае, когда изменения можно было бы просто добавить в ветку без создания дополнительного коммита.

Конфликт слияния веток может возникнуть, когда в двух ветках произошли изменения в одном и том же участке кода, которые не могут быть автоматически объединены. Для разрешения конфликта нужно вручную отредактировать файл(ы) с конфликтами, оставив только нужные изменения, а затем выполнить коммит с разрешением конфликта.

Ветку можно удалить командой:

php

git branch -d <branch>
где <branch> - название ветки, которую нужно удалить. Если ветка не была слита с другими ветками, нужно использовать опцию -D вместо -d.
